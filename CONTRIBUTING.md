# Contributing to HERO.Bot

Please read these guidelines before contributing.

Commit messages
- Use conventional commits style. Examples:
  - `feat: add new animation loader`
  - `fix: correct WebSocket binding`
  - `docs: update simulator README`

Branching
- Use feature branches named `feat/<short-desc>` or `fix/<short-desc>`.
- Keep `main` protected; create PRs for changes and assign reviewers.

Pull Requests
- Provide a clear description, link issues, and include testing steps.
- Keep PRs small and focused.

Code style
- Follow existing project style. Use linting where reasonable.

Testing
- Run unit tests and simulator locally before opening PRs.
